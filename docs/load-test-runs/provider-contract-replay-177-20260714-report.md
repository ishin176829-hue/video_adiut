## 异常与容错

本轮事件中记录到：

- 解析类重试事件 10 次；
- 字段校验类重试事件 1 次；
- 临时错误重试事件 4 次；
- 4 个任务出现过 429 痕迹，2 个任务出现过 504 痕迹，均最终完成；
- 3 个任务在契约重试耗尽后产生人工复核项；
- 没有任务因熔断、模型并发槽等待或数据契约错误进入失败终态。

人工复核降级只表示该时间段没有获得可靠的结构化视觉结果，不代表内容违规，也不能按通过处理。

## 本次修复

1. 模型响应的空内容、非 JSON、字段校验失败在有界重试耗尽后，生成带时间戳的人工复核项并继续整片汇总。
2. 契约错误默认尝试次数从 2 次提高到 3 次，以增加 Key 轮换自愈机会。
3. 任务创建时 PostgreSQL 改为强一致写入并最多重试 3 次；创建元数据未成功落库时不允许进入 Redis 队列。
4. PostgreSQL 后续状态 UPSERT 继续使用 `COALESCE` 保护已有 `request`，避免被无请求上下文的 worker 更新覆盖。

## 后续瓶颈

当前优先级不再是继续堆 worker，而是提升模型侧有效吞吐：

1. 将平均排队/额度等待从约 268 秒压缩到 60 秒以内。
2. 提升可用模型 QPM 或增加独立额度池，并按 Key 的成功率、延迟、429/504、契约合规率动态调度。
3. 将字幕通道和视觉批次进一步流水化，减少单视频各阶段串行等待。
4. 对 3 条人工复核样本保留原始模型响应诊断信息，继续降低空响应和非 JSON 比例。
